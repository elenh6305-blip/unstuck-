# 更新日志：增设后端 API 代理模型 (BYOK 支持)

**更新目标**：支持自动回退机制。用户自带 Key 时走前端直连，不填 Key 时走自有后端接口。同时兼容 Vercel Serverless 与独立服务器部署。

## 📍 核心逻辑
- **前端直连 (原有)**：如果检测到本地存有 API Key，直接请求 `geminiService`。
- **后端代理 (新增)**：如果没有 API Key，请求发往自己的后端接口 `/api/generate`（调用 DeepSeek）。
- **统一路由**：新增 `aiService.js` 作为核心调度枢纽。

## 📁 文件变更清单
### ✨ 新增 (5个)
1. `src/services/aiService.js`：调度器（有Key走直连，无Key走后端）。
2. `src/services/backendService.js`：用来向后端发送具体请求的客户端代码。
3. `api/generate.js`：真正的后端处理逻辑，请求 DeepSeek 的 API（Serverless 函数）。
4. `server/index.js`：为未来独立云主机部署准备的 Express 入口。
5. `vercel.json`：Vercel 的服务端路由与构建规则。

### 🔄 修改 (3个)
1. `src/hooks/useTaskBreakdown.js`：仅修改引用的服务为 `aiService`。
2. `vite.config.js`：新增开发环境 Proxy 代理，解决本地跨域。
3. `package.json`：新增后端运行必装依赖（`express`, `cors`, `dotenv`）。

### 📄 配置 (1个)
- `.env.example`：列出典型的环境变量模板，供本地后端运行参考。

## 🚀 部署兼容性
此结构下，前端代码（React）实现零修改适配多端部署：
- 推送到 Vercel 时，Vercel 原生识别 `api/generate.js`。
- 放进云服务器时，`server/index.js` 直接挂载 `api/generate.js` 作为 Express 路由。
